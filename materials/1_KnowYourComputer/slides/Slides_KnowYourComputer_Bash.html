2.1 Bash

Open Terminal

  • For Linux Ubuntu: you can type Ctrl-Alt-T or you can type Terminal into the Search box.
  • For Windows: open Git Bash
  • For Mac OS X: the Terminal.app that you will find in the Applications → Utilities folder.

2.2 Git

Github

In the terminal, once you are in the location where you want to download EnExDi2023 materials, type:
git clone https://github.com/ABC-DH/EnExDi2023.git

How to get the link: go to
bit.ly/EnExDi2023, click the green button "Clone or download" and copy the link that will appear.

Alternative (if git not installed): go to
bit.ly/EnExDi2023, click the green button "Clone or download", choose Download Zip. Unzip it and place where you want in your computer.

Github

Pull remote changes

...if the owner of the repository has changed/added/deleted any file...


1. Move to the EnExDi2023 directory:
cd EnExDi2023
2. Pull the remote changes:
git pull

Good practice (suggested)

At the beginning of every course:

  1. Open your terminal
  2. Navigate to EnExDi2023, For example: 
    cd EnExDi2023
  3. Type 
    git pull
    to update the folder.

Alternative (if git not installed): go to
https://github.com/ABC-DH/EnExDi2022, click the green button "Clone or download", choose Download Zip. Unzip it and place where you want in your computer (and delete the old version!).

File naming conventions

  • No space characters or punctuation other than hyphens (-), underscores (_), and periods (.) should be used in filenames
  • Choose a capitalization strategy and observe it consistently throughout your project
  • Filenames that contain multiple logical words can avoid the use of spaces and still maintain legibility by using underscores, hyphens, or camel case, e.g., “index_places.txt”, “index-places.txt”, or “indexPlaces.txt”. Choose a convention and use it consistently
  • If you keep multiple versions or drafts of files in the same directory, include a date stamp in the filename. We recommend a year-month-date format, with a four-digit year and two-digit months and dates, e.g., “2016-07-30_index-places.txt”.
  • Filenames should have a meaningful filename extension, like .xml, .txt, .docx
